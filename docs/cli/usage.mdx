# Cline CLI Usage Guide

A comprehensive guide to using Cline's command-line interface for AI-powered coding assistance.

## Installation & Setup

### Prerequisites
- Node.js 18 or higher
- npm or yarn package manager
- Terminal with modern features (iTerm2, Windows Terminal, etc.)

### Installation
```bash
# From the repository
cd cline-cli/cli
npm install
npm run build

# Link for global usage
npm link
```

### Initial Configuration
```bash
# Set your AI model and API key
cline config --model claude-3-sonnet
cline config --api-key your-api-key-here

# Initialize in a project directory
cd your-project
cline init
```

## Command Line Interface

### Basic Commands

#### Interactive Chat Mode (Default)
```bash
# Start interactive session
cline

# Start with specific context
cline --context "web development project"

# Resume previous session
cline --resume
```

#### Single Command Mode
```bash
# Execute one-off commands
cline "implement user authentication with JWT"
cline "refactor this component to use hooks"
cline "add tests for the payment module"

# With file context
cline "fix the bug in src/auth.ts" --file src/auth.ts
```

#### Configuration Commands
```bash
# View current configuration
cline config

# Set AI model
cline config --model claude-3-sonnet
cline config --model gpt-4
cline config --model gemini-pro

# Set API key
cline config --api-key sk-your-key-here

# Set temperature (creativity level)
cline config --temperature 0.7

# Enable verbose logging
cline config --verbose true
```

#### Project Management
```bash
# Initialize Cline in current directory
cline init

# View session status
cline status

# View chat history
cline history

# Clear chat history
cline clear

# Export conversation
cline export --format json --output conversation.json
```

## Advanced Usage

### Working with Large Projects

#### Project Context
```bash
# Analyze entire codebase
cline "analyze the architecture of this project"

# Focus on specific directories
cline "review the components in src/ui/" --path src/ui

# Exclude certain files
cline "refactor the API layer" --exclude "*.test.js,node_modules"
```

#### File Management
```bash
# Process multiple files
cline "update all React components to use TypeScript" --pattern "*.jsx"

# Backup before changes
cline "refactor authentication system" --backup

# Dry run mode
cline "optimize database queries" --dry-run
```

### AI Model Optimization

#### Model Selection Guidelines
- **Claude Sonnet**: Best for complex reasoning and code generation
- **Claude Haiku**: Faster responses for simple tasks
- **GPT-4**: Strong general programming capabilities
- **GPT-3.5 Turbo**: Cost-effective for basic tasks

#### Temperature Settings
- **0.0-0.3**: Deterministic, focused responses
- **0.4-0.7**: Balanced creativity and accuracy
- **0.8-1.0**: High creativity for brainstorming

### Session Management

#### Conversation Persistence
```bash
# Save conversation
cline export --format markdown --output session-$(date +%Y%m%d).md

# Load previous session
cline resume --session session-20241201.json

# Set session name
cline --session-name "api-development"
```

#### Context Management
```bash
# Set project context
cline context --add "This is a React TypeScript project using Vite"

# Add file context
cline context --file src/types.ts

# Clear context
cline context --clear
```

## Troubleshooting

### Common Issues

#### API Key Problems
```bash
# Verify API key is set
cline config | grep apiKey

# Test API connection
cline "test connection" --verbose

# Reset API key
cline config --api-key ""
```

#### Performance Issues
```bash
# Enable verbose logging
cline config --verbose true

# Check system resources
cline status --system

# Clear cache
cline cache --clear
```

#### Display Problems
```bash
# Check terminal compatibility
echo $TERM

# Force color mode
cline --color always

# Disable animations
cline config --animations false
```

### Error Messages

#### Authentication Errors
- **"Invalid API key"**: Check your API key configuration
- **"Rate limit exceeded"**: Wait before making more requests
- **"Model not available"**: Verify model name spelling

#### File System Errors
- **"Permission denied"**: Check file/directory permissions
- **"File not found"**: Verify file paths are correct
- **"Disk full"**: Check available disk space

#### Network Errors
- **"Connection timeout"**: Check internet connectivity
- **"SSL certificate error"**: Update certificates or use `--ignore-ssl`

## Tips and Best Practices

### Effective Prompting
1. **Be Specific**: "Add error handling to the login function" vs "fix the code"
2. **Provide Context**: Include relevant file names and line numbers
3. **Iterative Refinement**: Build on previous responses
4. **Use Examples**: Show desired output format when possible

### Project Organization
1. **Initialize Early**: Run `cline init` in new projects
2. **Regular Sessions**: Use consistent session names for ongoing work
3. **Export Important Conversations**: Save breakthrough solutions
4. **Context Management**: Keep relevant files in context

### Performance Optimization
1. **Choose Appropriate Models**: Use faster models for simple tasks
2. **Limit Context Size**: Focus on relevant files only
3. **Use Caching**: Enable response caching for repeated queries
4. **Monitor Usage**: Track API usage and costs
